# Quickstart

Get the SJSU Purity Test up and running on your local machine in just a few minutes.

<Steps>
  <Step title="Clone the repository">
    Clone the project from GitHub and navigate to the project directory:

    ```bash
    git clone https://github.com/dannymang/sjsuricepurity.git && cd sjsuricepurity
    ```
  </Step>

  <Step title="Install dependencies">
    Install the project dependencies using your preferred package manager:

    <CodeGroup>
      ```bash npm
      npm install
      ```

      ```bash yarn
      yarn install
      ```

      ```bash pnpm
      pnpm install
      ```
    </CodeGroup>
  </Step>

  <Step title="Set up environment variables">
    Create a `.env.local` file in the root directory with your MongoDB connection string:

    ```bash
    MONGODB_URI=your_mongodb_connection_string
    ```

    The app requires a MongoDB connection to store test results and power the community statistics feature. The connection string format for MongoDB Atlas looks like:

    ```
    mongodb+srv://<username>:<password>@cluster.mongodb.net/<dbname>?retryWrites=true&w=majority
    ```

    <Note>
      You'll need a MongoDB Atlas account (free tier available) or a local MongoDB instance running on your machine. If you're using MongoDB Atlas, make sure to whitelist your IP address in the cluster's network access settings.
    </Note>
  </Step>

  <Step title="Run the development server">
    Start the Next.js development server:

    <CodeGroup>
      ```bash npm
      npm run dev
      ```

      ```bash yarn
      yarn dev
      ```

      ```bash pnpm
      pnpm dev
      ```
    </CodeGroup>

    This starts the development server with Turbopack enabled for fast hot module replacement.
  </Step>

  <Step title="Open the app">
    Open [http://localhost:3000](http://localhost:3000) in your browser to see the SJSU Purity Test in action.

    You can now:
    - Answer the 100 questions about your SJSU experiences
    - Get your purity score
    - View community statistics

    <Tip>
      The app is fully functional even without MongoDB! The purity score is calculated entirely client-side in `src/app/page.tsx`. If the API call to save results fails, the error is caught and you're still redirected to the results page with your score. Only the community statistics feature requires a working database connection.
    </Tip>
  </Step>
</Steps>
